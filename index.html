Data structures, down to the cache line and out to the cluster.

Most books stop at Big-O. This one shows what the machine actually does — cache misses, memory layout, branch predictors — and how the systems you use every day, from Redis to OpenSearch, turn these structures into infrastructure that scales.

From the cache line to the cluster

Each structure is followed from what the CPU does with it — cache, memory layout, branch prediction — up to how it shards across a fleet of machines. The span no other DSA book covers.

Measured, not asserted

The benchmarks run in your browser, on your CPU, while you read. No screenshots of someone else's laptop, and no numbers you have to take on faith.

Connected to production

Every chapter ends where the structure actually ships — Redis, Postgres, the Linux kernel, OpenSearch shard routing — in modern C++ that compiles, with the invariants and failure modes spelled out in every chapter.
